This will be a demonstration and example of why box-sizing: border-box is the GOAT of CSS, ok maybe not the goat but top 10 CSS properties for sure

INTRO

To understand box-sizing we need to understand the box model, aka everything is a box, or box inside of a box, boxes in boxes in more boxes.. like a nesting doll; only more finicky and Angular ;)

By default box-sizing

content
scale
border-box

Let's now duplicate the above, but this time add 25px to content-box div

content
scale
border-box

Now our content-box div is +25 pixels on all four sides, and it's right side is sticking way out there, and this is the default behaviour

Now let's add box-sizing: border-box to our border-box div above

content
scale
border-box

So by using border-box, we include the padding and the border together, it makes things easier when styling, it will automatically do the math for you, it generally makes things much more predictable

Setting a width or height with border-box makes it so that padding and border are AUTO set up for you

ALSO REMEMBER TO ADD:

*, *::before, *::after { box-sizing: border-box; }

This is critically important because the universal selector by itself '*' WILL NOT select pseudo-elements, so rmemeber to add the above at the top of your CSS file

Alternatively you could, and may often see this in other people's code, we can do this to achieve the same effect:

*, *::before, *::after{box-sizing: inherit}

html{ box-sizing: border-box; }

When we use inherit with '*' and the before & after pseudo-selectors it can prevent some headaches when working with others code, (or my own code lol let's be real here)
